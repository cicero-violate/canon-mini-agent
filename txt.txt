## Math Model

[
\text{State Space}' = \text{State Space} \cap V_{inv}
]

### Variables

* (S): total possible states
* (V_{inv}): invariant-valid states
* (I(s)): invariant predicate (true/false)
* (A): actions
* (T): transitions

---

## Equations

1. **Invariant Gate**
   [
   I(s) \in {0,1}
   ]
   → state allowed or rejected

2. **Transition Filtering**
   [
   T'(s \rightarrow s') = T(s \rightarrow s') \cdot I(s')
   ]
   → illegal transitions never execute

3. **Action Constraint**
   [
   A' = {a \in A \mid I(\text{result}(a)) = 1}
   ]
   → only valid actions exist

4. **Canonical Collapse**
   [
   s_1 \sim s_2 ;\text{if}; \forall I,; I(s_1)=I(s_2)
   ]
   → merge states by invariant equivalence

---

## How It Works

1. **Define invariants (hard rules)**

* example:

  * no_actionable_failure ⇒ cannot repair
  * missing_target ⇒ must plan
  * validation_blocked ⇒ cannot verify

2. **Move invariants to runtime (not post-check)**

* enforce at:

  * route selection
  * planner output
  * action execution

3. **Reject invalid states early**

* instead of:

  * generate → verify → fail
* do:

  * **never generate invalid path**

4. **Collapse equivalent states**

* if two states produce same invariant results → treat as one

---

## Implementation (Concrete)

### Step 1 — Encode invariants

* central engine (you already have: `ConstraintState`)
* each invariant = pure function:
  [
  I: s \rightarrow {0,1}
  ]

---

### Step 2 — Gate routing

* replace:

  * heuristic routing
* with:
  [
  \text{Route} = \arg\max (I(s_{next}))
  ]

---

### Step 3 — Constrain planner

* planner must output:
  [
  \forall a,; I(\text{result}(a)) = 1
  ]
* reject plans violating invariants immediately

---

### Step 4 — Enforce at writer (you already do partially)

* invalid transition:
  [
  T(s \rightarrow s') = 0
  ]
  → emit violation, do not append

---

### Step 5 — State hashing (optional but powerful)

[
\text{Hash}(s) = \text{Hash}(V_{inv}(s))
]
→ identical invariant states collapse automatically
